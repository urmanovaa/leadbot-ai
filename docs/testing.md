# Тестовые сценарии LeadBot AI

## Подготовка

1. Убедитесь, что `.env.local` содержит все переменные
2. Сгенерируйте embeddings-индекс: `npm run build:embeddings`
3. Запустите проект: `npm run dev`
4. Откройте http://localhost:3000 и нажмите на иконку чата

---

## Сценарий 1. Полный flow — горячий лид

**Цель:** Пройти все 5 стадий от greeting до completed.

1. Откройте чат
2. Напишите: «Хочу заказать чат-бота для интернет-магазина»
3. Дождитесь уточняющих вопросов, отвечайте на них
4. Когда попросят контакт, напишите: «@myusername»
5. Подтвердите согласие на обработку данных

**Ожидаемый результат:**
- Стадии переключаются: greeting → qualification → contact_request → consent_request → completed
- Лид сохранён в Google Sheets со всеми полями
- Telegram-уведомление отправлено
- В консоли — логи всех стадий и событий

---

## Сценарий 2. Вопрос из базы знаний

**Цель:** Проверить, что RAG находит релевантный контекст.

1. Откройте чат
2. Напишите: «Что такое LeadBot AI?»

**Ожидаемый результат:**
- Ответ содержит информацию из базы знаний (01_project_overview.md)
- В консоли: `rag_context_found` с method embeddings или keyword

---

## Сценарий 3. Вопрос о стоимости

**Цель:** Проверить обработку вопроса о цене.

1. Откройте чат
2. Напишите: «Сколько стоит такой бот?»

**Ожидаемый результат:**
- Ответ объясняет, что цена зависит от задачи
- Не называет точных цифр (или ориентиры из базы знаний)
- Задаёт уточняющий вопрос

---

## Сценарий 4. Вопрос вне темы

**Цель:** Проверить fallback для нерелевантных вопросов.

1. Откройте чат
2. Напишите: «Какая погода в Москве?»

**Ожидаемый результат:**
- Бот мягко возвращает к теме AI-ассистентов
- Не придумывает ответ о погоде

---

## Сценарий 5. Отказ от контакта

**Цель:** Проверить обработку отказа.

1. Пройдите квалификацию до стадии contact_request
2. Напишите: «Не хочу оставлять контакт»

**Ожидаемый результат:**
- Бот предлагает продолжить консультацию
- Не давит на пользователя
- Стадия не переключается на completed

---

## Сценарий 6. Отказ от согласия

**Цель:** Проверить обработку отказа от согласия.

1. Дойдите до стадии consent_request
2. Нажмите кнопку «Отказ»

**Ожидаемый результат:**
- Бот не сохраняет контакт
- Предлагает продолжить консультацию
- Лид НЕ отправляется в Google Sheets
- Telegram-уведомление НЕ отправляется

---

## Сценарий 7. Просьба передать менеджеру

**Цель:** Проверить прямой запрос передачи менеджеру.

1. Откройте чат
2. Напишите: «Передайте меня менеджеру»

**Ожидаемый результат:**
- Бот не затягивает, задаёт 1-2 минимальных вопроса
- Запрашивает контакт и задачу

---

## Сценарий 8. Телефонный контакт

**Цель:** Проверить парсинг телефона.

1. Дойдите до стадии contact_request
2. Напишите: «+7 999 123 45 67»

**Ожидаемый результат:**
- Контакт распознан как phone
- Стадия переключается на consent_request

---

## Сценарий 9. Keyword fallback (без embeddings)

**Цель:** Проверить работу без embeddings-индекса.

1. Переименуйте `knowledge_base/.cache/embeddings.json` в `embeddings.json.bak`
2. Перезапустите сервер
3. Задайте вопрос из базы знаний

**Ожидаемый результат:**
- Ответ всё равно использует базу знаний
- В консоли: `rag_fallback_keyword`
- После теста: верните файл обратно

---

## Сценарий 10. Ошибка API

**Цель:** Проверить graceful degradation.

1. Установите невалидный OPENAI_API_KEY в .env.local
2. Перезапустите сервер
3. Отправьте сообщение

**Ожидаемый результат:**
- Пользователь видит сообщение об ошибке
- В консоли: `openai_request_failed` с описанием ошибки
- Приложение не падает

Проверьте, что в таблице появляются все колонки:

- [ ] createdAt
- [ ] name
- [ ] contact
- [ ] contactType
- [ ] task
- [ ] business
- [ ] goal
- [ ] timeline
- [ ] leadStatus
- [ ] consentGiven
- [ ] summary
- [ ] source
- [ ] rawConversation
- [ ] telegramSent

- [ ] Уведомление приходит в чат
- [ ] Содержит имя, контакт, задачу, статус
- [ ] При ошибке Telegram — лид всё равно сохраняется в Sheets
